# Code of Conduct

## Our Pledge

We want `codex-plugin-cc` to be a practical, respectful, security-conscious
open source project. Contributors and maintainers are expected to participate
with care, clarity, and professional judgment.

This Code of Conduct is adapted from Contributor Covenant version 2.1.

## Expected Behavior

Examples of behavior that helps the project:

- Be respectful of different backgrounds, experience levels, and viewpoints.
- Give technical feedback with evidence and a path to improvement.
- Keep security reports private until a fix and disclosure plan are ready.
- Accept responsibility for mistakes and correct them promptly.
- Focus discussion on the project, its users, and the safety of the release
  process.

## Unacceptable Behavior

Unacceptable behavior includes:

- Harassment, threats, intimidation, stalking, or sustained disruption.
- Insults, slurs, sexualized language or imagery, or unwanted sexual attention.
- Publishing private contact details, credentials, proprietary code, regulated
  personal data, or other confidential information.
- Abusing issues, pull requests, reviews, or security reports to attack people
  rather than improve the project.
- Any conduct that maintainers reasonably judge harmful to a professional open
  source community.

## Scope

This Code of Conduct applies in project spaces, including GitHub issues, pull
requests, discussions, reviews, release artifacts, and official project
communication. It also applies when someone represents the project in public.

## Enforcement

Maintainers may remove comments, close issues, reject contributions, restrict
participation, or report abuse to GitHub when needed. Enforcement decisions
will consider severity, impact, prior behavior, and the safety of affected
people.

Security-sensitive reports are handled through the private advisory process in
`SECURITY.md`.

## Reporting

Report Code of Conduct concerns to the maintainer through GitHub private
contact channels where possible. Do not include credentials, private user data,
or exploit details in public issues.
